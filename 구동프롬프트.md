# 구동 프롬프트 — 하이닉스 수급-주가 분석 자동화

아래 내용을 Claude Code(또는 Cowork)에 그대로 붙여넣고, 그 아래에 CSV 파일들을 첨부하면 된다.

---

## [붙여넣을 프롬프트]

너는 `analyze_supply.py`를 이용해 수급-주가 분석 리포트를 생성하는 작업을 한다.

### 입력 파일
내가 보통 키움 HTS에서 뽑는 두(또는 세) 종류의 CSV를 준다:
1. **가격 데이터** — 일자별 OHLC(종가/시가/고가/저가/거래량 등). 파일명에 "가격" 또는 "종가"가 들어가는 경우가 많다.
2. **수급 데이터(본주)** — 투자자별 순매수(개인/외국인/기관계/금융투자/투신/연기금 등). 파일명에 "수급"이 들어가는 경우가 많다.
3. **(선택) 선물 수급 데이터** — 주식선물 투자자별 순매수. 파일명에 "선물"이 들어간다. 있으면 외국인 현물×선물 매트릭스와 변동폭 분석까지 나온다. 없으면 본주 분석만 수행된다.

### 형식이 매번 다를 수 있으니, 실행 전에 반드시 확인할 것
raw data는 뽑을 때마다 형식이 조금씩 다르다. 스크립트를 돌리기 **전에** 다음을 점검하고, 어긋나면 스크립트가 아니라 *입력 인자나 전처리*로 맞춰라(스크립트 본문은 가능한 한 건드리지 말 것):

1. **어느 파일이 가격/수급/선물인지**를 컬럼을 열어보고 판별한다. 파일명만 믿지 말고 헤더를 확인하라. (가격=시가·고가·저가 있음 / 수급=개인·외국인·금융투자 등 투자자명 있음 / 선물=투자자명 + 파일/헤더에 '선물')
2. **인코딩**은 보통 cp949(euc-kr)지만 utf-8일 수도 있다. 스크립트가 자동 감지하나, 깨지면 알려라.
3. **헤더가 1줄인지 2줄인지** 확인한다. 키움 수급 데이터는 헤더가 2줄로 나뉘어 컬럼명이 아래 줄에 있는 경우가 잦다(예: '금융투자','연기금등'이 둘째 헤더 줄에 위치). 스크립트는 컬럼명 기반으로 위치를 찾으므로 대응되지만, 만약 컬럼명이 비표준(예: '연기금'이 '연기금등'이 아니라 다른 표기)이면 알려라.
4. **날짜 구분자**가 `/`인지 `-`인지 확인한다(스크립트가 양쪽 처리).
5. 컬럼 순서가 표준과 크게 다르거나 컬럼명이 없으면, 먼저 몇 행을 출력해 나에게 보여주고 매핑을 확인받은 뒤 진행하라.

### 실행
점검이 끝나면 다음 형태로 실행한다(파일명은 실제 파일로 교체):

```
python analyze_supply.py \
  --price  <가격CSV> \
  --flow   <수급CSV> \
  --futures <선물CSV>   # 선물 데이터가 있을 때만. 없으면 이 줄 생략
  --out    수급분석_리포트.md
```

### 실행 후
1. 생성된 `수급분석_리포트.md`를 나에게 보여준다.
2. 0번 상관표에서 **외국인 본주가 양(+)인지, 나머지 주체가 대체로 음(−)인지**를 확인하라. 만약 부호가 과거와 크게 다르면 컬럼 매핑 오류일 수 있으니 점검하라(특히 연기금/사모펀드 위치 혼동).
3. 표만 기계적으로 옮기지 말고, **눈에 띄는 변화나 직전 분석과 달라진 점**이 있으면 짧게 짚어라. 단, 매매 추천·목표가·진입시점·현재가 코멘트는 하지 마라(아래 원칙).

### 분석 원칙 (반드시 지킬 것)
- 결과물 맨 위와 맨 아래에 다음 취지를 **반드시 명시**한다(스크립트가 이미 넣지만, 네 코멘트에서도 빠뜨리지 말 것):
  > **이 분석은 내가 투자 판단에 쓰는 정보 중 아주 일부일 뿐이다.** 수급-가격의 과거 통계적 관계를 검증·반증하는 보조 도구이며, 펀더멘털·거시·외생변수·실시간 정보는 포함하지 않는다. 미래 예측치나 매매 신호가 아니다.
- **하지 말 것:** 매수/매도/타이밍 추천, 목표가·가격 전망, 현재가·등락 코멘트, 외국인·기관 매수세 강도에 대한 실시간 해석(터미널에서 내가 직접 본다), "투자는 본인 책임" 류의 상투적 경고.
- **할 것:** 날짜 명시, 표의 수익 기준(당일 수급 → 익일/+N일) 명시, 표본 수(n) 명시, 가속 국면·단일 추세장 등 통계의 적용 한계 명시.
- 표본(n)은 참고용이다. 작은 표본의 칸도 수치로 그대로 두되, n이 작으면 변동이 크다는 점만 감안해 읽어라(별도로 칸을 빼거나 사례로 분리하지 말 것).
- 가속 국면(20일선 이격 극단, 일중변동 분포 끝) 데이터가 보이면, 그 구간엔 통계의 정상성이 깨져 예측력이 급감함을 명시하라.

---

## 메모 (사용자용)

- 스크립트는 선물 CSV가 없으면 자동으로 "본주만" 모드로 돌아간다(외국인 매트릭스·변동폭 분석은 생략, 0번 상관표·밴드 병렬 분석은 그대로 수행).
- 컬럼명 기반 파싱이라 헤더 위치가 바뀌어도 대체로 견딘다. 단 컬럼명 자체가 비표준이면 구동 프롬프트의 점검 4·5번에서 걸러진다.
- 다른 종목에도 그대로 쓸 수 있다(하이닉스 전용 로직 없음).
